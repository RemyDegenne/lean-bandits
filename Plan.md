#  Plan for the Lean Machine Learning repository

LML is a library of machine learning algorithms and theorems in Lean.
It serves as the hight quality foundation on which researchers build when formalizing their machine learning work.
When someone asks an LLM to formalize their latest paper, it should start by importing LML.

The LML repository will have 3 main user-facing parts:
1. A Lean project with code for machine learning definitions and theorems.
2. An automatically generated documentation
3. A Verso website, generated by a second Lean project in the repository, which demonstrates how to use LML to write Machine Learning proofs.

The development of the library will also use the following:
- a zulip channel for discussion
- a roadmap
- a blueprint, which serves as a detailed roadmap used to work collaboratively on formalization tasks
- github issues. "Large" issues to track progress in broad areas, and "small" issues to dispatch work on individual lemmas

## Library organization

Top folders, at least at the beginning:
- ForMathlib - auxiliary lemmas that should be up-streamed
- Concentration - concentration inequalities (those might get up-streamed to Mathlib as well)
- Algorithms - general tools to describe algorithms
- Optimization - optimization theory and algorithms (gradient descent, etc.)
- Online - online convex optimization, bandits, experts
- Reinforcement - reinforcement learning
- Supervised - PAC learning, statistical learning theory
- Statistical - VC dim, Rademacher complexity, etc.

Add folders (after discussion) as needed. Missing from the list above: neural network theory, kernel methods, and much more.

## Verso website

The website contains tutorials and how-to guides about the library.
We write pages in which we explain how to do specific tasks.
Example: how to build a bandit algorithm and prove a regret bound for it.

What it is not:
- documentation of all results. We already have doc-gen for that.
- a formalization guide for the library. It's not a blueprint.

## Short and medium term targets

- tutorial: define and analyze a bandit algorithm
- library: optimization algorithms (stochastic gradient descent)
- tutorial: define SDG and prove an error bound
- library: online learning
- library: PAC learning

## Code quality and LLM policy

The strength of the library lies in carefully designed and reviewed definitions.
The standard is reusable code, not merely code that compiles.
We want to build the basis on which others can base their research efforts.

LLM use is not discouraged, but their output should be carefully reviewed for code quality before submitting pull requests.
